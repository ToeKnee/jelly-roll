Requirements

Satchmo requires a recent (0.95 or higher) version of Django. Until we reach 1.0, we encourage you to use the latest svn version of Django.

To get the latest version of Django: svn co http://code.djangoproject.com/svn/django/trunk/

You must now configure Django & get it working on your system. Follow the instructions here - http://www.djangoproject.com/documentation/install/

After Django is working, you can check out the latest version of Satchmo, using this command - svn co http://svn.satchmo.python-hosting.com/trunk/

To use the image functions, you will need a recent copy of the Python Imaging Library (PIL) available here - http://www.pythonware.com/products/pil/#pil115

For the timezone functions you will need pytz - World Time Zone Definitions for python available here - http://pytz.sourceforge.net

The category functions use ElementTree to make processing easier. It will be included in Python 2.5. You can also download it here - http://effbot.org/zone/element-index.htm

Since we're still putting this project together, you willl more than likely be running this in the development server. In order to ease the check-in and check-out process, we've modified the settings.py file so that some of the key information is read from a local file that is not controlled by svn. In order for everything to work right on your machine, you should copy the local_settings-customize.py to local_settings.py

Then, edit local_settings.py to have you database specific information:

DATABASE_NAME = 'Your DB Name Here'
DATABASE_PASSWORD = 'Your DB Password Here'
DATABASE_USER = 'Your DB User Here'
SECRET_KEY = 'Your secret key her'

Next, install your databases:

cd /path/to/satchmo
python manage.py syncdb

Once the database is installed, try the development server:

python manage.py runserver

Point your browser to localhost:8000 and you should see the admin interface

Several scripts are included to load temporary data or localization data. To see how Satchmo works, try loading the test data - WARNING this will erase your existing Satchmo database.:

python /path/to/satchmo/load_data.py

To load the localization data:

python /path/to/satchmo/load_G11n.py

You can point your browser to localhost:8000/shop to see the default shop.

Go to localhost:8000/admin to see the admin interface.

If you want to use the email feature, you'll need to make sure you have a shop email configured in the admin interface.

Helpful Hints

Here are two shell aliases I use to make it a little easier to develop:

alias delpyc='find ./ -type f -name "*.pyc" -exec rm -f {} \;'
alias startserver='python manage.py runserver 192.168.1.9:8000'

The delpyc is helpful if you want to clear out all your .pyc files and make sure there isn't any weirdness with old files.

The startserver just makes it easier to start up the development server with a given IP.

The load_data.py script is used to dump your current database and install some sample data. Please make sure you know what you're doing before you dump your database - the data will be lost!
